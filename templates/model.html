PINet Model Description

“PINet”, which stands for Point Instance Network, is an end-to-end framework, which predict key points and cluster those points to each instance, representing each traffic lane. Therefore, it can predict an arbitrary number of lanes rather than a fixed number.

PINet was designed adopting two stacked hourglass modules and with three output branches to predict the location of each point and clustering those points together. Moreover, the model is further applied a post-processing step to eliminate outlier of proposed points in order to reduce the false positive rate and smooth each lane.

Involved Modules

The most important part of the network: Hourglass module.

The hourglass module looks like an hourglass and it is usually applied to key points estimation tasks.
The first part of hourglass, from top to down, known as “encoder”, is used to extract feature from the input image and down-sampling to get high-level feature maps.
And the second part, known as “decoder,” is to up-sample the feature maps from bottom to up and restore them to the original resolution.
Meanwhile, there are some skip connections from encoder to decoder. This structure can make better use of information at every scale and combine local features together with global features.
